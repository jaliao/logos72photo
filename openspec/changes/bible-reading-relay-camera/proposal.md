## Why

為了支援 72 小時不間斷的讀經接力活動，我們需要一個自動化的定時拍照與照片管理系統。由於參與者名單難以精確掌握，我們決定將系統改為**全開放模式**，讓所有參與者能根據拍照時段自由檢閱與下載照片。

## What Changes

- **自動化定時拍照**：建立一個系統，能夠每 5 分鐘自動觸發兩台 iPhone 進行拍照。
- **全開放照片下載**：移除參與者身分驗證機制，改為按時段開放瀏覽與下載。
- **時段化照片管理**：
    - 每日分為三個主要時段（相簿集）：00-08, 08-16, 16-24。
    - 每個主要時段內，再細分為每 15 分鐘一個子相簿（例如 08:00-08:15）。
- **照片檢索介面**：使用者可透過簡單的時段選單，快速定位並瀏覽對應的 15 分鐘相簿。
- **多裝置支援**：系統需同時管理並同步兩台 iPhone 的拍照任務。
- **系統監控與即時回饋**：確保現場工作人員能掌握裝置狀態，並在拍照時獲得視覺回饋。

## Capabilities

### New Capabilities
- `camera-control`: 遠端或自動觸發行動裝置相機拍照的機制。
- `photo-storage-management`: 儲存照片並與時間戳記、裝置 ID 進行關聯，並支援按時段索引。
- `photo-retrieval-ui`: 提供按「8 小時時段」與「15 分鐘子相簿」層級瀏覽與下載照片的公開介面。
- `monitoring-dashboard`: 監控裝置狀態（電量、連線、最後拍照時間）並提供操作回饋。

### Removed Capabilities
- `participant-management`: 移除外部 API 驗證與參與者時段查詢功能。

## Impact

- **資料庫 (Firebase)**: 使用 Firestore 存儲照片 metadata，需針對時段查詢進行索引優化。
- **儲存 (Cloudflare R2)**: 存放所有拍攝的照片實體。
- **部署 (Cloudflare Pages)**: 託管前端應用程式。
- **身分驗證**: 移除對外部報到系統的依賴。
- **API 路由**: 簡化為僅處理照片 metadata 查詢與 R2 下載連結。
