## Context

`AlbumPhotoViewer` 目前在 expand 模式（照片展開後）顯示單行授權說明：「本照片用於活動行銷宣傳，如不同意請點「刪除」自行移除。」語氣較生硬，且 grid 模式（縮圖列表）完全沒有任何說明，導致家人不確定照片用途。

## Goals / Non-Goals

**Goals:**
- 以更友善的多段文字取代現有單行通知
- 在 grid 模式頂部加入同一份說明橫幅，讓家人進入相簿即可看到

**Non-Goals:**
- 不修改刪除流程或任何 API 行為
- 不加入「同意」按鈕或任何互動確認機制

## Decisions

### 決策 1：grid 模式橫幅放置位置

**選擇：** 照片網格上方、卡片容器內頂部。

**理由：** 家人打開相簿後第一眼即看到說明，不需點進任何照片。放在卡片容器內可與現有 `bg-white/50 rounded-2xl` 外框整合，視覺一致。

### 決策 2：expand 模式維持現有通知區塊位置

**選擇：** 維持放在操作按鈕上方的位置，更新文字內容。

**理由：** 位置已為家人熟悉（v0.1.53 起已存在），只需更新文字，不變動 UX 結構。

### 決策 3：多段文字格式

**選擇：** 以 `<p>` 或換行分隔三段，保持可讀性。

**理由：** 原始文字為三個獨立句子，分段展示比連成一行更易閱讀，手機尤其明顯。

## Risks / Trade-offs

- **文字過長影響版面**：多段文字在小螢幕可能占據較多空間 → 緩解：使用 `text-sm`，字體不過大，三段說明在 320px 寬螢幕約佔 5–6 行，可接受。
