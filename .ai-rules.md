# AI 指令規範 (Gemini CLI / OpenSpec)

## 一、 文件生成規範 (README-AI.md)
當執行變更或要求更新上下文時，必須依序填充以下章節：
1. **專案核心目標 (Core Objective)**：簡述系統價值。
2. **技術棧 (Tech Stack)**：須標註精確版本（Next.js 16.1.1, Prisma 7.2.0）。
3. **系統架構 (System Architecture)**：說明 App Router 與 Data Access Layer 的分工。
4. **核心資料模型 (Data Schema)**：描述 `prisma/schema/` 中的實體關聯。
5. **關鍵業務邏輯 (Business Logic)**：如 pgcron 自動化、Timer 跨分頁同步、CR 管理流程。
6. **開發規範 (Coding Standards)**：強制 ActionResponse 格式、繁體中文標頭與註解。
7. **當前挑戰與任務 (Current Status & Backlog)**：反映當前處理中的 CR 編號。

## 二、 掃描與 Token 優化準則 (重要)
- **執行前必讀**：在讀取任何檔案前，必須先透過 `ls` 檢查結構，並嚴格排除 `.claudignore` 中定義的路徑。
- **最小化上下文**：若任務僅涉及「邏輯」，禁止讀取 `app/` 下的 UI 頁面組件以節省 Token。
- **增量輸出**：代碼變更僅輸出「受影響的區塊」。使用 `// ... existing code` 標記未修改部分。
- **配置優先**：列舉值與常數必須優先參考 `config/*.ts`。

## 三、 自動化工作流
- **版本控制**：每次執行代碼變更套用（apply），必須自動將 `config/version.json` 的 patch 版本號 +1。
- **同步更新**：變更完成後，必須依照此規範同步更新 `README-AI.md`，確保版本號與技術現狀一致。
